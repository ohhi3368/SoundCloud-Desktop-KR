# Гайд по тулзе редактирования файлов (для агента)

## Имя тулзы

`edit_file` — именно так, не `edit`, не `multi_edit`, не `str_replace`.

## Параметры (camelCase, не snake_case)

| Параметр     | Тип     | Назначение                                       |
| ------------ | ------- | ------------------------------------------------ |
| `TargetFile` | string  | Абсолютный путь к файлу (обязателен **первым**)  |
| `oldString`  | string  | Текст, который надо заменить (точное совпадение) |
| `newString`  | string  | Новый текст                                      |
| `replaceAll` | boolean | Опционально, заменить все вхождения              |

## Правила

1. **Сначала `TargetFile`**, потом остальные параметры — иначе schema-ошибка.
2. **`oldString` должен быть уникален** в файле. Если не уникален — добавить контекст вокруг или использовать `replaceAll: true`.
3. **Точное совпадение пробелов/табов/переносов**. Если читал через `view_file` — копировать содержимое после префикса `N: ` (номер строки + двоеточие + пробел в этом окружении).
4. **`oldString` ≠ `newString`** — иначе ошибка no-op.
5. **Перед первым редактом файла обязательно прочитать его** через `view_file`.
6. **Создание нового файла** — через `write_to_file` (`TargetFile` + `CodeContent` + `EmptyFile: false`), а не через `edit_file` с пустым `oldString`.

## Параллельные правки

Несколько `edit_file` к **разным файлам** — можно в одном сообщении параллельно.
Несколько правок одного файла — тоже можно параллельно, но `oldString` не должны пересекаться. При сомнении — последовательно.

## Типичные ошибки в этом окружении

- `SchemaError(Missing key at ["filePath"])` → передал тулзе пустые аргументы или забыл `TargetFile`.
- `SchemaError(Missing key at ["oldString"])` → передал только `TargetFile`, забыл `oldString`/`newString`.
- `old_string not found` → проверить отступы и невидимые символы; читать файл через `view_file` без переформатирования.

## Шаблон вызова

```
edit_file(
  TargetFile = "/абсолютный/путь/файла.ts",
  oldString  = "точный кусок старого текста",
  newString  = "новый текст"
)
```

## Альтернативы

- `view_file` — чтение (поддерживает `StartLine`/`EndLine`).
- `write_to_file` — создание нового файла. **Не использовать для перезаписи существующего.**
- `grep_search` — поиск по содержимому.
- `find_by_name` — поиск по имени/glob.
- `run_command` — shell, **только** для git/build/package manager, не для поиска и не для эха.
